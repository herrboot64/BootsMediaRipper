# Case Study: Boots Media Ripper

## Problem

Creating media screenshots, short video clips, and audio snippets manually was repetitive and error-prone. The workflow required finding files, handling paths, remembering ffmpeg syntax, formatting timestamps, avoiding bad filenames, and organizing output afterward.

## Solution

Boots Media Ripper wraps that workflow in a local Windows tray/server app, browser UI, HTTP API, ffmpeg integration, and Android companion client.

## Why this is portfolio-worthy

- Local-first application design
- API-backed UI workflows
- Windows scripting and packaging
- Android companion app behavior
- ffmpeg automation
- File, path, and timestamp handling
- Logging and troubleshooting
- Security-minded public repo preparation
- AI-assisted development with human-owned direction and testing
